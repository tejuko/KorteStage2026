## Wat heb ik geleerd?
- `winning_case` CSS staat **dubbel** (inline fragment + `dynamic-seo.css`) → altijd beide bijwerken, anders zie je je wijziging niet terug.
- Generieke regels (`.dynseo-section p {color:#000}` + Open Sans op `<strong>`) overschreven mijn sectie-styling → `!important` nodig om er overheen te komen.
- RUCSS-cache vereist een **DB-row delete** (niet alleen `rocket_clean_domain`) om echt te verversen.
- Verifiëren via headless render + CDP computed-styles, niet alleen de CSS-tekst lezen — dat is hoe je zeker weet dat de browser het écht zo rendert.
- ACF echt leek-proof maken (instructies bij elk veld, jargon eruit) is wat de "dummyproof"-eis uit de briefing concreet maakt.
